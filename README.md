# Telegram-бот: погода + курс валют (5 языков)

Бот показывает актуальную погоду и курс валют, интерфейс переключается
на 5 языков: 🇷🇺 русский, 🇬🇧 английский, 🇺🇿 узбекский, 🇰🇷 корейский, 🇸🇦 арабский.

Используются бесплатные API без ключей:
- **Погода** — [Open-Meteo](https://open-meteo.com/)
- **Курс валют** — [open.er-api.com](https://www.exchangerate-api.com/)

## Файлы проекта

```
tgbot/
├── bot.py              — основной код бота
├── translations.py     — все тексты на 5 языках
├── requirements.txt    — зависимости Python
├── .env.example        — пример файла с токеном
├── Dockerfile           — для запуска в Docker
├── tgbot.service        — для запуска как systemd-службы (24/7 на VPS)
└── users_lang.json      — создаётся автоматически, хранит выбранный язык каждого пользователя
```

## 1. Получить токен бота

1. Откройте в Telegram [@BotFather](https://t.me/BotFather)
2. Отправьте `/newbot`, придумайте имя и username бота
3. Скопируйте выданный токен (выглядит как `123456789:AAExample...`)

## 2. Запуск локально (для теста)

```bash
cd tgbot
python3 -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
pip install -r requirements.txt

cp .env.example .env
# откройте .env и вставьте свой токен вместо примера

python bot.py
```

Если всё правильно — в консоли появится `Бот запущен, ожидаю сообщения...`.
Откройте бота в Telegram и нажмите /start.

## 3. Запуск на 24/7

Локальный компьютер для этого не годится (бот "умрёт" при выключении/сне ПК).
Есть три простых варианта:

### Вариант A — Dockerдешёвый VPS (рекомендую)

Подойдёт любой VPS от 1$/мес (Timeweb, Hetzner, Vultr, Beget, Selectel и т.д.).

```bash
# на сервере, после загрузки туда папки tgbot
cd tgbot
cp .env.example .env   # и вписать токен
docker build -t weather-currency-bot .
docker run -d --name tgbot --restart always --env-file .env weather-currency-bot
```

`--restart always` — контейнер сам поднимется после перезагрузки сервера.
Проверить логи: `docker logs -f tgbot`

### Вариант B — systemd на VPS (без Docker)

```bash
cd tgbot
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
cp .env.example .env   # вписать токен

# отредактируйте tgbot.service — замените USERNAME на ваше имя пользователя и путь
sudo cp tgbot.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable tgbot
sudo systemctl start tgbot

# проверить статус и логи
sudo systemctl status tgbot
sudo journalctl -u tgbot -f
```

Бот будет сам перезапускаться при падении и после перезагрузки сервера.

### Вариант C — облачные платформы (без своего сервера)

Подойдут Railway, Render, Fly.io — есть бесплатные лимиты для старта.
Загрузите туда репозиторий (например, через GitHub), укажите переменную
окружения `BOT_TOKEN`, команду запуска `python bot.py` — и платформа сама
держит бота включённым 24/7.

## Если бот не подключается к Telegram (ConnectTimeout)

Если сам Telegram (приложение/web.telegram.org) открывается нормально, а бот
выдаёт `httpx.ConnectTimeout` / `telegram.error.TimedOut` — значит ваш
провайдер блокирует именно адрес `api.telegram.org` (Bot API), которым
пользуется код бота, хотя само приложение Telegram работает через другие
серверы/протокол. Такое часто встречается у провайдеров в СНГ.

Решения (по возрастанию надёжности):

1. **Включить VPN на компьютере** перед запуском `python bot.py` — самый
   простой способ, если VPN уже есть.
2. **Запустить бота через прокси.** В `.env` добавьте строку:
   ```
   PROXY_URL=socks5://127.0.0.1:1080
   ```
   (адрес и порт зависят от вашего прокси/VPN-клиента — многие VPN-программы
   поднимают локальный SOCKS5- или HTTP-прокси, посмотрите в их настройках).
   Бот автоматически подхватит этот параметр.
3. **Запустить бота на VPS за границей** (см. раздел "Запуск на 24/7" выше) —
   там обычно проблем с доступом к Telegram нет вообще, и заодно бот будет
   работать 24/7.

## Возможности бота

- `/start` — запуск, выбор языка
- `/weather` или кнопка "Погода" — узнать погоду по названию города (или по сохранённому городу)
- `/forecast` или кнопка "Прогноз на неделю" — прогноз погоды на 7 дней
- `/currency` или кнопка "Курс валют" — курс USD/EUR к UZS, RUB, KRW, AED, UAH, CNY, KZT, GBP, TRY
- Просто напишите сумму (например `10$`, `100 евро`, `50000 сум`) — бот покажет её эквивалент в других валютах
- `/crypto` или кнопка "Крипто" — курс Bitcoin и Ethereum (в USD, RUB, UZS)
- `/mycity` или кнопка "Мой город" — сохранить город по умолчанию, чтобы не вводить его каждый раз
- `/notify ЧЧ:ММ` — включить ежедневную рассылку погоды (по сохранённому городу) и курса валют в указанное время (по Ташкенту, UTC+5)
- `/notify off` — выключить рассылку
- `/language` — сменить язык в любой момент
- `/help` — справка

### Как работает ежедневная рассылка

Рассылка использует встроенный планировщик задач python-telegram-bot (JobQueue,
на основе APScheduler) — поэтому в `requirements.txt` подключена зависимость
`python-telegram-bot[job-queue]`. При каждом перезапуске бота (например, после
редеплоя на Render) все сохранённые рассылки автоматически восстанавливаются
из файла `users_settings.json` — пользователям не нужно включать их заново.

Обратите внимание: время в `/notify` указывается по Ташкенту (UTC+5),
независимо от того, в каком часовом поясе находится сервер (Render обычно
использует UTC) — бот сам это учитывает.

## Как добавить ещё валюты или города по умолчанию

В `bot.py` есть список `TARGET_CURRENCIES` — добавьте туда код нужной
валюты (стандартные 3-буквенные ISO-коды, например `GBP`, `CNY`).

## Как добавить ещё язык

1. В `translations.py` добавьте новый код языка в `LANGUAGES`, `TEXT` и `WEATHER_DESC`
   (скопируйте структуру любого существующего языка и переведите значения).
2. Больше ничего менять не нужно — бот подхватит язык автоматически.
